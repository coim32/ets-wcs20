Compliance Test Suite for Web Coverage Service (WCS) 2.0.1

Overview

This executable test suite is based on the following OGC specifications:

  • OpenGIS® Web Coverage Service(WCS) 2.0 Interface Standard - Core (OGC 09-110r3)
  • OGC® WCS 2.0 Interface Standard- Core: Corrigendum (09-110r4)
  • OGC® GML Application Schema - Coverages (09-146r2)
  • OpenGIS® Web Coverage Service 2.0 Interface Standard - KVP Protocol Binding Extension (1.0) (OGC 09-147r1)
  • OpenGIS® Web Coverage Service 2.0 Interface Standard - XML/POST Protocol Binding Extension (1.0) (OGC 09-148r1)
  • OpenGIS® Web Coverage Service 2.0 Interface Standard - XML/SOAP Protocol Binding Extension (1.0) (OGC 09-149r1)
  • Compliance Test Language (CTL) 0.5

What is tested

The WCS 2.0 core is tested.

  • GetCapabilities (KVP, Post, SOAP)
  • DescribeCoverage (KVP, Post, SOAP)
  • GetCoverage (KVP) on GML coverage are tested

The extensions (KVP, Post, and SOAP) are only tested when the GetCapabilities document indicates that the extensions are supported.

Requirements

General requirements for the implementation:

  1. At least one protocol implementation should be available and advertised in the GetCapabilities
  2. The implementation provides at least two coverages.
  3. The first coverage offering should be RectifiedGridCoverage with at least 2 dimensions, while the second coverage could be any coverage.
  4. Do not use CoverageId_Bogus as a coverage ID, format_bogus as a format name, mediatype_bogus for the mediatype parameter, or dimension_bogus as an axis name.
  5. The offered coverage must be implementd according to GMLCov specification.
